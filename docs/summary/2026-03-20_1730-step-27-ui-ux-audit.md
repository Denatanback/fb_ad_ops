# Summary — Step 27: UI/UX Audit

## Status: Audit-only. No code changes.

---

## Core diagnosis

The product has a working design system and solid data model. The main problem is not visual style — it is **structural repetition** and **explanation overload**. Every page uses the same template and explains its own contents too much.

The UI acts like it's always the user's first visit. An internal operations tool should act like the user knows where they are and what the numbers mean.

---

## The 10 most important findings

1. **Triple title** — sidebar label + topbar h1 + PageHeader eyebrow/title/description. Three redundant title sources per page.

2. **SectionCard is the only card primitive** — used for filters, forms, tables, annotations, and nested containers alike. Nothing feels primary vs secondary.

3. **Stat cards over-explain** — every stat card has a `stat-copy` paragraph that re-describes the stat label. On an operational screen, these are noise.

4. **Import run detail is an unstructured wall** — 15+ sections, equal visual weight. Alert events (the most operationally relevant section) appears deep below pipeline diagnostics.

5. **Dashboard approach actions are broken** — "Открыть подходы" inside an approach card goes to the full approaches list, not to the specific approach.

6. **Sidebar nav items look like content cards** — borders, background fill, and 18px radius make them indistinguishable from content blocks.

7. **Language inconsistency** — nav items in English (Queue, Active, Scaling, Creatives, Approaches, Landers), eyebrow labels in English ("Creative details", "Imports"), backtick code references (`TELEGRAM_ALERTS_ENABLED`) in Russian operational text.

8. **Admin pages contain wrong content** — target-costs admin page shows historical spend data and top approaches. Settings pages should only contain settings.

9. **Creative detail dumps all metadata** — 10 URL fields listed even when 7 of them say "Не указан". Empty fields should be hidden by default.

10. **Gallery lacks filter controls** — no approach/status filter, no total count, mode switcher lacks active state differentiation.

---

## What should change structurally

| Current | Target |
|---|---|
| PageHeader (eyebrow + h2 + description) inside content | Single page title in topbar or as first line of content |
| SectionCard for everything | Differentiated primitives: working area / annotation panel / settings section |
| Stat cards with paragraph descriptions | Compact summary strip: label + number only |
| Filter controls in a named SectionCard | Flat filter/action bar directly below page title |
| Import run: all 15 sections visible | Progressive disclosure: summary → alerts → collapsible diagnostics |
| Sidebar nav items styled as cards | Simple link style: hover highlight + active accent |
