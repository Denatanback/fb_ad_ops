# UI Polish Pass

## Target pages

- `/(workspace)` dashboard
- `/creatives/gallery`
- `/creatives/[creativeId]`
- `/approaches`
- `/imports` and import-related analyzer surfaces
- `/admin/notifications`
- `/admin/analyzer-rules`
- `/admin/target-costs`
- `/settings`
- `/admin/google-drive`
- `/guide`

## Likely visual inconsistencies to inspect

- awkward wrapping of long Russian labels and helper text
- uneven spacing between page header, toolbar, summary strip and section cards
- summary/stat blocks that feel too tall or too cramped
- toolbar links and chips with inconsistent density
- section-card header rhythm varying too much between old and new surfaces
- creative detail / funnels / admin forms using new patterns but not yet fully aligned in spacing
- table and list sections that visually clash with the denser shell

## Polish strategy

- keep the current Phase 1/2 shell intact
- prefer global CSS refinements over new abstractions
- apply small page-level copy/alignment tweaks only where they remove obvious clashes
- preserve server component boundaries and existing flows
- improve scanability first: rhythm, header density, text wrapping, action placement

## How the skills shape this pass

- `web-design-guidelines`
  - focus on hierarchy, spacing rhythm, reduced noise and operational scanability
- `shadcn`
  - keep composition restrained and practical; improve existing primitives instead of inventing new UI systems
- `vercel-react-best-practices`
  - keep pages server-first, avoid turning polish into client-side sprawl, and keep styling/data concerns separated

## Non-goals

- no backend/domain changes
- no new dashboard logic
- no new analyzer/import features
- no Phase 3 navigation or IA expansion
- no fake browser-automation claims if runtime browser tooling is unavailable
